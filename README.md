## 📁 프로젝트 구조
```
weather-mlops/
├─ data/                        # 데이터만 모아두는 곳
│  ├─ raw/                      # API 원천 (그대로 저장)
│  └─ features/                 # 피처 테이블 스냅샷
├─ src/                         # 실제 코드 (한 줄기)
│  ├─          
├─ conf/
│  └─ config.yaml               # 키/경로/스테이션ID 같은 설정
├─ tests/
│  └─ test_features.py          # 피처 로직 단위 테스트 1~2개
├─ notebooks/                   # EDA 노트북(선택)
│  └─ eda.ipynb
├─ .env.example                 # KMA_API_KEY=... (실제 .env는 깃에 올리지 말기)
├─ requirements.txt             # 의존성
├─ README.md                    



```
